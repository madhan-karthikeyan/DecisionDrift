## Context

The database models need primary keys that are unique across tables, not sequentially guessable, and safe to expose in URLs and API responses.

## Decision

Use UUID v4 strings (`db.String(36)`) as primary keys for all models instead of auto-incrementing integers. UUIDs are generated at the application level using Python's `uuid.uuid4()` and stored as strings.

## Consequences

UUIDs prevent enumeration attacks (users cannot guess the next ID) and simplify data merging across databases. The 36-character string key has a minor storage and index performance cost compared to integer keys, but at the expected data volume this is negligible. Application-level generation means the ID is available before the first database write.
