## Context

The system needs to notify users through multiple channels: email for appointment confirmations and reports, SMS for reminders, and Google Chat for internal alerts. Each channel has different delivery mechanisms and failure modes.

## Decision

Centralize all notification logic in `backend/utils/notifications.py` with a consistent interface: each function takes recipient and content parameters, returns a boolean success indicator, and logs errors instead of raising exceptions. Notifications are dispatched asynchronously via Celery tasks in `backend/tasks/` to avoid blocking HTTP requests.

## Consequences

A single file contains all notification channel logic, making it easy to add, remove, or modify channels. The graceful degradation pattern (log and return False on failure) ensures a misconfigured or unavailable notification channel never breaks the main application flow. The tradeoff is that callers must check the return value if they need delivery guarantees.
