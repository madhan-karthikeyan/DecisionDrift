## Context

The system needs to run recurring background jobs: daily appointment reminders and monthly doctor performance reports. These should execute automatically on a schedule without manual intervention.

## Decision

Use Celery Beat with `crontab` schedules for periodic task execution. Reminders run daily at 7:00 AM IST, and monthly reports run on the 1st of each month at 6:00 AM IST. The Beat schedule is configured in `init_celery()` alongside the Celery app configuration.

## Consequences

Celery Beat provides reliable cron-like scheduling integrated with the existing Celery infrastructure. The schedule is defined in code and version-controlled. The main operational requirement is that the Celery Beat process must be running alongside the worker — if Beat goes down, scheduled tasks stop executing.
