## Context

The hospital management system needs a web framework to serve HTTP requests, handle routing, manage sessions, and integrate with various extensions for database access, authentication, and task processing.

## Decision

Use Flask as the primary web framework. Flask provides a lightweight, extensible foundation with a large ecosystem of extensions. The application follows the Flask application factory pattern (`create_app()`) for testability and configuration flexibility.

## Consequences

Flask's minimal core keeps the application simple, but extensions must be explicitly chosen and configured for ORM, authentication, migrations, and other concerns. The application factory pattern enables different configurations for development, testing, and production without global state.
