## Context

The Flask application needs different configurations for development, testing, and production. A module-level global app instance makes it difficult to test with different configurations or create multiple app instances.

## Decision

Use Flask's application factory pattern. The `create_app()` function in `backend/__init__.py` accepts a configuration object, creates the Flask instance, initializes extensions, registers blueprints, and returns the configured app. Entry points (`run.py`, `manage.py`, `celery_worker.py`, tests) call `create_app()` with the appropriate configuration.

## Consequences

Tests can create isolated app instances with test-specific configurations. Extensions are initialized only when the factory runs, avoiding circular import issues common with global app objects. The pattern requires that all components (extensions, models, routes) initialize lazily rather than at import time.
