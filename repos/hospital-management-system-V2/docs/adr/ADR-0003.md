## Context

The system has three user roles (admin, doctor, patient) requiring authenticated access to protected endpoints. A stateless authentication mechanism is needed to avoid server-side session storage and support horizontal scaling.

## Decision

Use JWT (JSON Web Tokens) via Flask-JWT-Extended for authentication. The user's role is embedded as an additional claim in the token at login. Protected endpoints use the `@jwt_required()` decorator, and role-specific access is enforced via custom decorators (`require_admin`, `require_doctor`, `require_patient`) that check the role claim.

## Consequences

JWT enables stateless authentication without server-side session storage. Tokens can be validated by any instance, simplifying horizontal scaling. The tradeoff is that tokens cannot be revoked server-side before expiry, so token expiry times must be short (15-30 minutes) with a refresh token strategy for long-lived sessions.
